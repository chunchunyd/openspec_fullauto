# 设计说明：Chat Part1 Step 05 流式事件投影与 Assistant 消息落地

## 目标

把 runtime 的回复事件投影成平台侧可消费的 chat 语义，并在任务成功完成后将 assistant 回复正式落地为消息历史中的一条消息。

## 边界

- `runtime` 继续负责生成事件和最终结果。
- 平台 API 负责：
  - 维护 owner-scoped 的 chat 事件读取边界
  - 把 runtime 事件投影为 chat 可消费的流式片段或等价状态
  - 在成功完成后落地 assistant 消息
- 本 step 不负责：
  - 手动重试入口
  - 通用事件总线平台化
  - push、未读或已读回执

## 投影规则

1. runtime 事件必须先进入平台侧受控锚点或等价受控投影层。
2. 对外返回的 chat 事件语义应围绕会话、消息和 task，而不是暴露 provider 私有实现细节。
3. owner-scoped 读取必须复用 `chat` 的访问边界，不允许跨用户消费他人的事件结果。

## 最终落地规则

1. runtime 成功完成后，平台必须只落地一条最终 assistant 消息。
2. 会话的最近消息摘要与最近活跃时间必须根据最终 assistant 结果更新。
3. 平台不得因为重复事件或乱序回流而生成多条重复的最终 assistant 消息。

## 失败说明

本 step 只要求为后续失败重试保留受控事件与状态锚点，完整失败重试入口留给下一步处理。
