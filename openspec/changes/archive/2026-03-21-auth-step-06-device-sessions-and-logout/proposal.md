# 变更提案：Auth Step 06 设备会话管理与退出登录

## 为什么要做

会话已经建立之后，用户还需要对自己的登录设备和当前登录态有最小控制权。

如果设备列表、会话移除和退出登录长期缺失，会导致：

- refresh token 虽然存在，但用户无法主动撤销旧设备
- 设备会话模型只是数据库占位，无法真正用于治理
- 后续异常登录排查缺少最基本的用户自助入口

因此需要把设备会话管理与退出登录单独落成一个 step。

## 本次变更包含什么

本次变更聚焦设备会话治理，范围包括：

- 查看当前账号的设备会话列表
- 移除指定设备会话
- 主动退出当前设备登录态
- 让被移除或已退出的会话不再能继续维持刷新

## 本次变更不包含什么

本次变更不包含以下内容：

- 更复杂的异常设备评分和风控模型
- 管理后台代替用户管理其会话
- 第三方登录设备同步

## 预期结果

完成后，项目应具备以下结果：

1. 用户可以查看自己的设备会话列表
2. 用户可以移除某个设备或退出当前设备
3. 被撤销会话的 refresh token 不再继续维持登录态
4. 会话模型真正具备治理价值，而不是仅服务签发

## 影响范围

本次变更主要影响：

- `apps/api/src/modules/auth`
- auth 会话治理与撤销逻辑
- `packages/api_contracts` 导出的设备会话相关接口 contract
